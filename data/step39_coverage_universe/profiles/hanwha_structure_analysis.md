## 3. 요약표 vs 상세표의 역할 분리

### A. 요약표 ("가입담보요약", pages 3-4)

**위치**: pages 3-4
**컬럼 구조**:
```
번호 | 가입담보 | 가입금액 | 보험료(원) | 납기/만기
```

**특징**:
- 번호순 단순 나열 (1, 2, 6, 10, 29, 33, 45, 47, ...)
- **번호가 연속되지 않음** (HANWHA 고유 특징)
- 카테고리 구분 없음
- 담보명 + 금액 + 보험료 + 기간만 표시
- 셀 병합 없음 (1 row = 1 coverage)
- 갱신형 담보는 "(갱신형)" 접미어로 표시

**역할**:
- ✅ **가입 여부 확정** (Universe Lock)
- ✅ **금액/보험료 확정**
- ❌ 보장 내용 설명 없음
- ❌ 조건/감액/면책 정보 없음

---

### B. 상세표 ("담보 및 보장내용 예시", pages 5-10)

**위치**: pages 5-10
**컬럼 구조**:
```
번호 | 가입담보 | 가입금액 | 보험료(원) | 납기/만기 | 보장(보상)내용
```

**특징**:
- 선택계약만 있음 (기본계약 없음)
- 보장내용 컬럼에 **서술형 긴 문장** 기재
- 보장개시일, 감액 조건, 면책 기간 명시
- 예시: "보장개시일(계약일로부터 90일이 지난날의 다음날)"
- 예시: "가입 후 1년미만시 가입금액의 50%지급"

**역할**:
- ✅ **보장 내용 설명**
- ✅ **조건/감액/면책 명시**
- ✅ **고객 질의 대응 필수 자료**
- ❌ Universe 정의 용도 아님

## 4. STEP 3.9-1에서 Universe 추출에 사용할 표

**결론: 요약표 "가입담보요약" (pages 3-4)**

### 이유

1. **Universe Lock 원칙**:
   - 가입설계서에서 계약자가 **실제 가입한 담보 목록**의 유일한 출처
   - 요약표는 "무엇을 가입했는가"를 명시하는 **계약 요약**

2. **상세표는 보강용**:
   - 상세표는 "어떻게 보장하는가"를 설명하는 **설명서**
   - Universe 추출 후, 정보 보강 단계에서만 사용

3. **1:1 매핑 보장**:
   - 요약표의 모든 담보가 상세표에도 존재
   - 번호(No.)로 1:1 매핑 가능
   - 추가 담보 없음, 누락 담보 없음

### STEP 3.9-1 추출 대상 확정

```yaml
extraction_target:
  table: "가입담보요약"
  location: pages 3-4
  columns_to_extract:
    - 번호
    - 가입담보
    - 가입금액
    - 보험료(원)
    - 납기/만기
```

## 5. 고객 질의 시 반드시 참조해야 하는 영역

### 시나리오별 참조 영역

| 질의 유형 | 참조 표 | 이유 |
|-----------|---------|------|
| "암 진단비에 가입했나요?" | 요약표 | Universe 확인 |
| "암 진단비 보장금액은?" | 요약표 | 가입금액 확인 |
| "암 진단비 보험료는?" | 요약표 | 보험료 확인 |
| "암 진단비 90일 대기기간 있나요?" | **상세표** | 보장개시일 확인 필수 |
| "1년 미만 가입 시 감액되나요?" | **상세표** | 감액 조건 확인 필수 |
| "유사암은 제외되나요?" | **상세표** | 제외 조건 확인 필수 |

### 고객 질의 처리 순서

1. **Universe 확인**: 요약표에서 담보 존재 여부 확인
2. **금액 확인**: 요약표에서 가입금액/보험료 확인
3. **조건 확인**: 상세표에서 보장내용/조건/감액/면책 확인

### Q2. 요약표만으로 담보 "존재 여부"를 확정할 수 있는가?

**A2**: **Yes**

- 요약표에 없으면 → Universe에 없음 (`out_of_universe`)

- Universe 정의에 연령대 변형 고려 불필요

### Q4. 상세표는 어떤 단계에서만 사용되는가?

## 7. HANWHA 가입설계서 고유 구조적 특징

### SAMSUNG / KB / DB와의 공통점

- ✅ 2-table 구조 (요약표 + 상세표)
- ✅ 요약표 = Universe 출처
- ✅ 상세표 = 고객 질의 대응용
- ✅ 1:1 매핑 보장
- ✅ 번호순 단순 나열 (카테고리 구분 없음) ← **KB/DB와 동일**
- ✅ 갱신형 담보 "(갱신형)" 접미어 표시 ← **KB/DB와 동일**

### HANWHA 고유 특징

- ✅ **담보 번호가 연속되지 않음** (1, 2, 6, 10, 29, 33, 45, ...)
  - SAMSUNG/KB/DB: 1, 2, 3, 4, 5, ... (연속)
  - HANWHA: 1, 2, 6, 10, 29, ... (불연속)
  - **이유 추정**: 상품 설계 시 일부 담보 번호를 예약하거나, 특정 번호 체계를 따름

- ✅ 상세표에 "기본계약" 구분 없음 (선택계약만 존재)
  - SAMSUNG/DB: 기본계약 + 선택계약 구분
  - HANWHA: 선택계약만

- ✅ 보험료 합계 별도 표시
