2. **상세표** (pages 9-14): "보장사항"
   - 피보험자 정보 헤더 + 구분 + 담보명 + **보장내용(서술형 긴 문장)** + 납입 및 만기 + 가입금액 + 보험료

   - **보장 조건의 유일한 출처**

3. **순수 텍스트 구간** (pages 6-7, 15-18)
   - 계약사항 안내 (pages 6-7)
   - 보험료 납입면제 안내 (page 15)
   - 보험가격지수 안내 (page 16)
   - 계약자 확인사항 길라잡이 (page 17)
   - 경과기간별 예상 해약환급금예시표 (page 18)

## 3. 요약표 vs 상세표 차이점

### 3-1. 요약표 ("가입담보 리스트", pages 7-8)

**컬럼 구조**:
```
구분 | 담보명 | 납입 및 만기 | 가입금액 | 보험료(원)
```

**특징**:
- 구분 카테고리: "기본" / "선택"
- 담보명 전체 기재 (단축 없음):
  ```
  기본  일반상해후유장해(80%이상)                                      20년납 100세만기  1,000만원  130
  선택  암진단비(유사암제외)                                           20년납 100세만기  3,000만원  35,730
  선택  재진단암Ⅲ진단비                                                20년납 100세만기  1,000만원  32,790
  ```
- **병합된 셀 없음** (LOTTE와 동일)
- 갱신형 담보 표시:
  - 담보명에 "[갱신형]" 접두어 명시
  - 납입 및 만기에 "10년갱신 100세만기" 기재
  ```
  선택  [갱신형]표적항암약물허가치료비Ⅱ(갱신형_10년)                   10년갱신 100세만기  1,000만원  80
  ```
- 보험료 합계 단일 표시:
  ```
  보장보험료 합계  137,353원
  ```
  (갱신/비갱신 분리 없음, LOTTE와 동일)

**역할**:
- "어떤 담보에 가입했는가"의 절대 기준
- Universe Lock의 SSOT (Single Source of Truth)

---

### 3-2. 상세표 ("보장사항", pages 9-14)

**컬럼 구조**:
```
피보험자(1/1) | 고객님(나이:30세, 성별:남자, 상해급수:상해1급, 운전여부:운전안함)
구분 | 담보명 + 보장내용 | 납입 및 만기 | 가입금액 | 보험료(원)
```

**특징**:
- **피보험자 정보 헤더 명시** (다른 보험사와 차별화):
  ```
  피보험자(1/1)  고객님(나이:30세, 성별:남자, 상해급수:상해1급, 운전여부:운전안함)
  ```

- 구분 카테고리 동일: "기본" / "선택"

- 보장내용 긴 서술형 문장:
  ```
  암진단비(유사암제외)  20년납 100세만기  3,000만원  35,730

  보험기간 중 보장개시일 이후 암(유사암제외)으로 진단확정시 금액 지급
  ※ 최초 1회, 계약일로부터 1년미만 발생시 가입금액의 50%지급
  ※ [보장개시일] 가입연령이 15세 미만인 피보험자 : 보험계약일 / 가입연령이 15세 이상인 피보험자 : 보험계약일로부터 그날을 포함하여 90일이 지난 날의 다음날

  [ 보험금을 지급하지 않는 사항 ]
  ①보통약관(보험금을 지급하지 않는 사항) 제1항에 따른 사항
  ```

- 주석(※)으로 조건 명시:
  - 보장개시일 (예: 암 90일 대기)
  - 감액 조건 (예: "1년 미만 50% 감액")
  - 제외 사항 (예: "유사암 제외")
  - 지급 한도 (예: "최초 1회한", "연간 1회한")

- **제외사항 표기 형식** (HEUNGKUK 고유 특징):
  ```
  [ 보험금을 지급하지 않는 사항 ]
  ①보통약관(보험금을 지급하지 않는 사항) 제1항에 따른 사항

  또는

  [ 보험금을 지급하지 않는 사항 ] 보통약관
  → 제1항 : ①피보험자가 고의로 자신을 해친 경우 ②보험수익자가 고의로 피보험자를 해친 경우 ...
  → 제2항 : ①전문등반, 글라이더 조종, 스카이다이빙, ... ②모터보트, 자동차 경기, ...
  ```

**역할**:
- "이 담보가 어떻게 작동하는가"의 절대 기준
- 고객 질의 응답 시 필수 참조 문서

## 4. 고객 질의와 요약표의 한계

### 4-1. 예시 질의 1

**고객 질문**: "암 진단비는 90일 대기 기간이 있나요?"

**요약표로는 답변 불가**:
- 요약표에는 "암진단비(유사암제외) 3,000만원 20년납 100세만기 35,730"만 기재
- 보장개시일 정보 없음

**상세표로만 답변 가능** (page 11):
```
암진단비(유사암제외)

보험기간 중 보장개시일 이후 암(유사암제외)으로 진단확정시 금액 지급
※ [보장개시일] 가입연령이 15세 이상인 피보험자 : 보험계약일로부터 그날을 포함하여 90일이 지난 날의 다음날
```

👉 **답변**: 네, 15세 이상은 90일 대기 기간이 있습니다.

---

### 4-2. 예시 질의 2

**고객 질문**: "뇌졸중 진단비는 1년 미만이면 감액되나요?"

**요약표**: "뇌졸중진단비 1,000만원 20년납 100세만기 8,890" (조건 정보 없음)

**상세표** (page 10):
```
뇌졸중진단비

보험기간 중 뇌졸중으로 진단확정시 금액 지급
※ 최초 1회, 계약일로부터 1년미만 발생시 가입금액의 50%지급
```

👉 **답변**: 네, 1년 미만이면 50% 감액됩니다.

---

### 4-3. 결론

**고객 질의 응답은 상세표 없이는 불가능**

요약표는 "무엇을 비교할 것인가" (Universe Lock)에만 사용
상세표는 "어떻게 해석할 것인가" (조건, 감액, 보장개시일)에 사용

## 5. 요약표와 상세표의 관계

### 5-1. 1:1 대응 여부

**기본 원칙**: 1:1 완전 대응

**검증**:
- ✅ 요약표 37개 담보 = 상세표 37개 담보
- ✅ 담보명 정확히 일치
- ✅ 구분(기본/선택) 카테고리 동일

**예외 케이스**: **없음**
- LOTTE와 동일하게 유사암 병합 없음
- 모든 담보가 1:1 대응

---

### 5-2. 요약표에만 있고 상세표에 없는 담보?

**없음**. 요약표의 모든 담보가 상세표에 존재함.

---

### 5-3. 상세표에만 있고 요약표에 없는 담보?

**없음**. 상세표는 요약표의 완전 확장판임.

## 6. STEP 3.9-1 Universe 추출 핵심 판단

### 6-1. Universe 추출 대상 표

**대상 표**: 요약표 "가입담보 리스트" (pages 7-8)

**근거**:
1. **Coverage Universe Lock 원칙 준수**:
   - 가입설계서에서 "고객이 실제 가입한 담보 목록"의 유일한 출처는 요약표
   - 상세표는 보장 조건만 추가 기재할 뿐, 가입 여부를 결정하지 않음

2. **SSOT (Single Source of Truth)**:
   - 요약표 = "What coverages are in this contract"
   - 상세표 = "How each coverage works"

3. **중복 회피**:
   - 요약표와 상세표를 모두 추출하면 동일 담보가 중복 추출됨
   - Universe는 단일 출처에서만 추출해야 함

4. **HEUNGKUK 구조적 장점**:
   - 요약표에 병합 셀 없음 (LOTTE와 동일, SAMSUNG보다 파싱 용이)
   - 구분(기본/선택) 카테고리로 명확히 식별 가능
   - 담보명 전체 기재 (단축 없음)

---

### 6-2. 상세표의 역할

**상세표는 Universe 추출 대상이 아님**.

**역할**:
- Universe 추출 후, 각 담보의 조건·감액·보장개시일을 **보강(Augmentation)** 하는 용도
- 고객 질의 응답 시 RAG retrieval 대상 (compare_axis, policy_axis)

**처리 방법**:
1. STEP 3.9-1: 요약표에서 담보명 + 가입금액 + 보험료 추출 → Universe 테이블
2. STEP 후속: 상세표에서 보장내용 + 조건 추출 → Augmentation 또는 별도 테이블

---

### 6-3. 변형 파일 처리 방식

**질문**: 성별/연령 변형 파일이 Universe 정의에 영향을 주는가?

**답변**: **해당 없음** (변형 파일 존재하지 않음)

**처리 방법**:
